# NanoAgent 技术栈学习路线

这份路线面向想参与 NanoAgent 开发的人，按当前项目实际技术栈和代码路径组织。建议边读文档边跑项目，而不是只看概念。

## 0. 先跑起来

目标：能在本机启动、构建并定位主要目录。

需要掌握：

- Windows 下 `npm.cmd`、Node.js、Rust toolchain、WebView2、MSVC Build Tools。
- `npm.cmd run tauri dev`、`npm.cmd run build`、`cargo check` 的用途。
- Tauri app data 目录和三份 SQLite 数据库的作用。

建议阅读：

- [README](../README.md)
- [构建、配置与运维](构建配置与运维.md)

练习：

1. 启动开发模式。
2. 新建一个普通会话和一个项目会话。
3. 跑一次 `npm.cmd run build`。

## 1. 前端基础路线

目标：能读懂 UI、状态和后端调用之间的关系。

学习顺序：

1. TypeScript：接口、联合类型、可选字段、泛型返回值。
2. React：函数组件、props、state、effect、memo。
3. Vite：开发服务器、构建产物、模块导入。
4. CSS：项目现有组件样式、设置页、聊天区和 Ops 面板布局。

项目入口：

- `src/App.tsx`
- `src/components/ChatPane.tsx`
- `src/components/settings/SettingsModal.tsx`
- `src/components/settings/SettingsArchiveTab.tsx`
- `src/api.ts`

练习：

1. 从 `ChatPane` 追踪一条消息如何渲染。
2. 从 `SettingsArchiveTab` 追踪归档消息预览如何调用 `renderMessageContent`。
3. 修改一个只影响 UI 的小文案或布局，并用 `npm.cmd run build` 验证。

## 2. Tauri 与 Rust 后端路线

目标：能新增或修改一个 command，并理解安全边界。

学习顺序：

1. Rust 基础：所有权、Result、Option、match、错误传播。
2. Serde：前后端数据结构序列化。
3. Tauri command：`#[tauri::command]`、`State<'_, AppState>`、`invoke` 参数命名。
4. Tokio：异步函数、Mutex、外部进程。
5. 文件路径安全：根目录解析、相对路径归一化、越界防护。

项目入口：

- `src-tauri/src/lib.rs`
- `src-tauri/src/models.rs`
- `src-tauri/src/error.rs`
- `src/api.ts`

练习：

1. 找到 `read_chat_image_attachment` 的前后端调用链。
2. 找到一个 command 的 `models.rs` 类型、`api.ts` 封装和 UI 调用点。
3. 修改后同时跑 `npm.cmd run build` 和 `cargo check`。

## 3. SQLite 与本地数据路线

目标：能理解业务数据、运行时数据和观测数据为什么分库。

学习顺序：

1. SQLite 表、索引、事务、WAL。
2. FTS5 全文检索。
3. rusqlite 参数绑定和 row mapping。
4. 轻量 migration：`CREATE TABLE IF NOT EXISTS` 和 `ensure_column`。

项目入口：

- `src-tauri/src/db.rs`
- `src-tauri/src/runtime.rs`
- `src-tauri/src/observability.rs`

建议阅读：

- [数据与存储设计](数据与存储设计.md)

练习：

1. 追踪 `conversations` 和 `messages` 的创建、归档、删除。
2. 追踪 `agent_runs`、`agent_steps`、`agent_tool_calls` 的写入时机。
3. 追踪 `observability_spans` 如何按 trace 展示。

## 4. LLM、流式输出与 Agent 路线

目标：能理解一次用户消息如何变成模型请求、工具调用和续写。

学习顺序：

1. OpenAI-compatible `/chat/completions` 与 `/embeddings`。
2. Anthropic Messages API。
3. SSE/streaming 事件解析。
4. prompt/system message 组装。
5. XML tool call 协议和用户审批链路。

项目入口：

- `src/hooks/useChat.ts`
- `src/lib/chatSystemMessage.ts`
- `src/lib/messageHelpers.ts`
- `src-tauri/src/llm.rs`
- `src-tauri/src/agent_runner.rs`
- `src-tauri/src/runtime.rs`

建议阅读：

- [Agent、RAG、MCP 与 Skills](智能体检索增强与扩展工具设计.md)

练习：

1. 追踪 `chat_stream` 如何把 delta 发回前端。
2. 让模型输出一个 `read_file` tool call，观察审批和结果回写。
3. 查看 Agent Runtime 面板中的 run/step/tool call。

## 5. RAG 与 Embedding 路线

目标：能维护对话级 RAG 的索引和召回。

学习顺序：

1. 文本抽取和清洗。
2. chunk 策略。
3. embedding 向量和相似度。
4. FTS 与向量结果混合召回。

项目入口：

- `src/hooks/useRagFiles.ts`
- `src-tauri/src/db.rs`
- `src-tauri/src/llm.rs`

练习：

1. 上传一个文本文件并观察 RAG 文件状态。
2. 找到 `rag_files`、`rag_chunks`、`rag_embeddings` 的写入代码。
3. 修改 query 后观察召回片段变化。

## 6. OCR 图片附件路线

目标：能维护图片保存、预览、归档预览和 OCR 识别链路。

学习顺序：

1. 浏览器 File/base64 和系统拖拽路径。
2. Tauri 后端文件保存。
3. data URL 图片预览。
4. PaddleOCR CLI 调用和资源限制。

项目入口：

- `src/hooks/useChat.ts`
- `src/lib/appHelpers.tsx`
- `src/components/ImageAttachmentMessage.tsx`
- `src/components/settings/SettingsArchiveTab.tsx`
- `src-tauri/src/lib.rs`

建议阅读：

- [PaddleOCR OCR 工具](图片文字识别工具.md)

练习：

1. 在普通会话上传图片，确认图片保存到 app data `temp/`。
2. 在项目会话上传图片，确认图片保存到项目 `.nano-agent/uploads/images/`。
3. 归档会话后到 Settings > Archive 检查图片仍可预览和打开。

## 7. MCP 与 Skills 路线

目标：能理解外部工具如何进入模型上下文并被执行。

学习顺序：

1. MCP JSON-RPC、tools/list、tools/call。
2. stdio、SSE、streamable HTTP 的差异。
3. Skills 元数据、本地目录和参数替换。
4. 工具风险分级和用户审批。

项目入口：

- `src/hooks/useMcp.ts`
- `src/hooks/useSkills.ts`
- `src-tauri/src/mcp.rs`
- `src-tauri/src/skills.rs`
- `src-tauri/src/agent_runner.rs`

练习：

1. 配置一个 stdio MCP server 并连接。
2. 查看 MCP 工具如何注入 system message。
3. 启用一个 Skill，观察 `workspace_root` 和 `output_dir` 如何替换。

## 8. Ops 与观测路线

目标：能排查 SSH、模型、MCP 和工具调用问题。

学习顺序：

1. SSH 基础：host、port、username、key、agent、SFTP。
2. Tauri event 驱动终端输出。
3. trace/span 观测模型。
4. 非阻塞观测设计。

项目入口：

- `src/components/OpsPanel.tsx`
- `src-tauri/src/lib.rs`
- `src-tauri/src/observability.rs`
- `src/hooks/useObservability.ts`

练习：

1. 新增一个 Ops server 并测试连接。
2. 打开观测面板，按 trace 查看一次操作。
3. 触发一次失败连接，检查 error 和 metadata。

## 9. 推荐进阶任务

按难度递增：

1. 修复一个纯 UI bug，并保持 `npm.cmd run build` 通过。
2. 给已有 command 增加一个可选字段，并更新 `models.rs`、`api.ts` 和 UI。
3. 给归档预览或消息渲染补一个新消息类型。
4. 新增一个只读 Agent 工具。
5. 扩展一个观测 span 的 metadata，但避免记录敏感信息。
6. 为 RAG 召回调优 chunk 或排序策略。
7. 增加一个新的模型 provider 或 MCP transport。

## 10. 工作习惯

- 先从真实代码路径追踪，不靠猜测。
- 前端改动至少跑 `npm.cmd run build`。
- 后端 command、数据库、文件系统、模型链路改动同时跑 `cargo check`。
- Windows 命令优先使用 `npm.cmd`。
- 不提交 app data、数据库、构建产物、依赖目录和项目内 `.nano-agent` 附件目录。
- 涉及路径、安全、工具执行和密钥时，把边界写进文档和代码。
